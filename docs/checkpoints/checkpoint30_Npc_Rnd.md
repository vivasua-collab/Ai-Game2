# Checkpoint 30: Система временных NPC ("Статисты")

## Дата: 2026-03-01

## Статус: В разработке

---

## Цель

Реализовать систему временных NPC, которые:
- Существуют только в памяти сессии
- Генерируются при входе в локацию
- Удаляются при выходе или смерти
- Используют формулы Lore для характеристик

## Ключевые решения

### 1. Хранение данных

**Решение:** Memory Map с singleton паттерном

```typescript
// sessionId -> locationId -> TempNPC[]
private npcs: Map<string, Map<string, TempNPC[]>>
```

**Альтернативы:**
- ~~Redis~~ - избыточно для одной сессии
- ~~Prisma~~ - противоречит концепции временных данных
- ~~worldState JSON~~ - накладные расходы на сериализацию

### 2. ID генерация

**Решение:** `TEMP_` префикс + timestamp + random

```typescript
id: `TEMP_${Date.now().toString(36)}_${Math.random().toString(36).slice(2, 6)}`
```

**Преимущества:**
- Легко отличить от персистентных NPC
- Уникальность в рамках сессии
- Не конфликтует с БД ID

### 3. Интеграция с генераторами

**Решение:** Переиспользование существующих генераторов

```typescript
import { generateNPC } from '@/lib/generator/npc-generator';
import { generatedObjectsLoader } from '@/lib/generator/generated-objects-loader';
```

**Преимущества:**
- Переиспользование кода
- Единые формулы Lore
- Согласованность данных

### 4. Ответы на вопросы из random_npc.md

| Вопрос | Решение |
|--------|---------|
| Лут | Дроп из быстрых слотов + случайная генерация |
| Сохранение | Не сохраняем, генерируем заново при входе |
| Именные NPC | Через персистентные NPC (Prisma) |
| Агрессия | Зависит от роли (combat = агрессивные) |
| Боевые слоты | Как у игрока (3 + уровень) |
