### Требование пользователя
> "Мир игры и все взаимодействия полностью просчитываются на стороне сервера, клиент только отображение."

> "Постоянно живущий самостоятельный сервер и сессионные подключения игрока"

## 📋 План реализации

### Фаза 1: Серверный World State (2-3 часа)

**Задача:** Расширить TruthSystem для хранения мира

**Изменения:**
1. Создать `WorldState` интерфейс (NPCs, Locations, Events)
2. Расширить TruthSystem для работы с WorldState
3. Добавить методы управления NPC в памяти

**Файлы:**
- `src/lib/game/world-state.ts` - НОВЫЙ
- `src/lib/game/truth-system.ts` - расширить

### Фаза 2: WebSocket Service (2-3 часа)

**Задача:** Создать WebSocket сервис для real-time

**Изменения:**
1. Создать mini-service на порту 3003
2. Базовые события: connect, disconnect, world:sync
3. Интеграция с TruthSystem

**Файлы:**
- `mini-services/game-ws/index.ts` - НОВЫЙ
- `src/lib/game-socket.ts` - НОВЫЙ (универсальный клиент)

### Фаза 3: Серверный AI Manager (3-4 часа)

**Задача:** Перенести SpinalController на сервер

**Изменения:**
1. Создать серверный `NPCAIManager`
2. Адаптировать `SpinalController` для сервера
3. Tick Loop обновления NPC

**Файлы:**
- `src/lib/game/ai/server/npc-ai-manager.ts` - НОВЫЙ
- `src/lib/game/ai/server/spinal-server.ts` - НОВЫЙ
- `mini-services/game-ws/index.ts` - добавить AI tick

### Фаза 4: Клиентский Action Executor (1-2 часа)

**Задача:** Клиент только выполняет команды от сервера

**Изменения:**
1. Удалить SpinalController из NPCSprite
2. Добавить слушатель WebSocket событий
3. Визуальное выполнение действий

**Файлы:**
- `src/game/objects/NPCSprite.ts` - упростить
- `src/game/scenes/LocationScene.ts` - добавить WebSocket обработку

### Фаза 5: Тестирование (1-2 часа)

**Тесты:**
1. NPC реагирует на приближение игрока
2. NPC уклоняется от атаки
3. NPC атакует игрока при агрессии
4. NPC бежит при низком HP
5. Синхронизация при reconnect

## 📊 Анализ задержек

### При 1 TICK = 1 СЕКУНДА

| Режим | WebSocket задержка | Пригодность |
|-------|-------------------|-------------|
| Sandbox (Caddy) | 50-110 мс | ✅ 110мс << 1000мс |
| Localhost | 10-20 мс | ✅ Идеально |

**Вывод:** WebSocket обеспечивает достаточную скорость для AI при правиле 1 TICK = 1 СЕКУНДА
