## 📊 Архитектурный анализ

### Проблемы текущей архитектуры

| Проблема | Описание |
|----------|----------|
| **Sync vs Async** | Phaser синхронный (60 FPS), EventBus асинхронный (HTTP) |
| **Лаг отклика** | 50-200ms на каждое действие (HTTP round-trip) |
| **Дублирование** | Состояние на клиенте + сервере + API |
| **Сложность** | Много слоёв абстракции |

## 🔧 НОВЫЕ ИСПРАВЛЕНИЯ (18.03.2026 - продолжение)

### Урон по Training NPCs — ИСПРАВЛЕНО ✅
- **Проблема:** Атака проверяла только `globalTargets` (чучела), но не `globalTrainingNPCs` (монстры)
- **Решение:** Добавлен цикл проверки NPC в `executeTechniqueInDirection()`
- **Range базовой атаки:** Увеличен с 1м до 3м (разумная дистанция рукопашной)

### Техники при медитации — РАБОТАЕТ ✅
- Логи подтверждают: `[INFO] [GAME] Technique granted via meditation`
- 1-я длинная медитация (4+ часа): melee_strike техника
- 2-я длинная медитация: ranged техника

### Назначение техник в слоты — ИСПРАВЛЕНО ✅
- **Проблема:** `type` перезаписывался `subtype` (type='melee_strike' вместо type='combat')
- **Решение:** 
  - `technique-granter.ts`: `type: tech.type`, `subtype: tech.subtype`
  - `api/technique/slot/route.ts`: добавлены `support`, `sensory`, `healing` в список боевых типов
