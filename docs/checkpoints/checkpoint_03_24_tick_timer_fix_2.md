## 🔴 Сообщённые проблемы

1. **При движении время в React НЕ идёт** - остаётся на паузе
2. **Время в Phaser ИДЁТ** - цифры меняются
3. **На старте: React = 06:00, Phaser = 07:00** - разные значения
4. **Нет привязки скорости движения от времени** - время не влияет на скорость
5. **Нет привязки скорости боя от времени** - кулдауны не зависят от time speed

**Архитектурный анализ перенесён в:** [docs/TIME_SYSTEM.md](../TIME_SYSTEM.md)

## ✅ РЕШЕНИЯ ПОЛЬЗОВАТЕЛЯ

### 1. Разная скорость для разных действий?
**Ответ: ДА**

| Действие | Режим времени | Обоснование |
|----------|--------------|-------------|
| Путешествие (выход с локации) | `fast` / `ultra` | Ускорение перемещения |
| Исследование локации | `normal` | Стандартная скорость |
| Бой | `superSuperSlow` | Тактический контроль |
| Медитация | `ultra` | Максимальное ускорение |

### 2. Должна ли медитация (ultra) делать всё мгновенным?
**Ответ: ДА**

- В режиме медитации время ускоряется до `ultra` (60 мин/тик)
- Все действия масштабируются соответственно
- Игрок "проживает" часы за секунды

### 3. Нужен ли "боевой режим" времени?
**Ответ: ДА**

- Автоматическое переключение на `superSuperSlow` при начале боя
- Возврат к предыдущей скорости после боя
- Возможность ручного переключения во время боя

### 4. Многопользовательский режим?
**Ответ: НЕТ (пока)**

- Игра однопользовательская
- При появлении мультиплеера: все живут в одном времени со статичной скоростью тиков
- Клиент-серверная архитектура сейчас — для будущего перехода на другой графический движок

## 🏗️ АРХИТЕКТУРА TIME SCALING

### Принцип работы

```
┌─────────────────────────────────────────────────────────────────┐
│                    FUNDAMENTAL RULE                             │
│              1 TICK = 1 SECOND REAL TIME (FIXED)                │
│       Variable: minutesPerTick = game minutes per tick          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    TIME SPEED CONFIG                            │
├─────────────────┬───────────────┬───────────────────────────────┤
│ Speed           │ Min/Tick      │ Real → Game Ratio             │
├─────────────────┼───────────────┼───────────────────────────────┤
│ superSuperSlow  │ 0.25 (15 sec) │ 1 сек = 15 сек игры           │
│ superSlow       │ 0.5 (30 sec)  │ 1 сек = 30 сек игры           │
│ slow            │ 1 мин         │ 1 сек = 1 мин игры            │
│ normal          │ 5 мин         │ 1 сек = 5 мин игры            │
│ fast            │ 15 мин        │ 1 сек = 15 мин игры           │
│ ultra           │ 60 мин        │ 1 сек = 1 час игры            │
└─────────────────┴───────────────┴───────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                 SCALING FACTORS                                 │
│                                                                 │
│  Speed Multiplier = 5 / minutesPerTick                         │
│  (нормализовано к normal = 5 мин/тик)                          │
│                                                                 │
│  superSuperSlow: 5 / 0.25 = 20x (медленнее)                    │
│  superSlow:      5 / 0.5  = 10x (медленнее)                    │
│  slow:           5 / 1    = 5x  (медленнее)                    │
│  normal:         5 / 5    = 1x  (базовая)                      │
│  fast:           5 / 15   = 0.33x (быстрее)                    │
│  ultra:          5 / 60   = 0.083x (быстрее)                   │
└─────────────────────────────────────────────────────────────────┘
```

### Формулы масштабирования

#### 1. Скорость движения
```typescript
// BASE_SPEED = скорость при normal time
// scaledSpeed = BASE_SPEED * (5 / minutesPerTick)

const scaledSpeed = baseSpeed * TIME_SCALING_FACTOR[speed];
```

| Speed | Factor | Пример: BASE=200 |
|-------|--------|------------------|
| superSuperSlow | 20.0 | 4000 px/sec |
| superSlow | 10.0 | 2000 px/sec |
| slow | 5.0 | 1000 px/sec |
| normal | 1.0 | 200 px/sec |
| fast | 0.33 | 66 px/sec |
| ultra | 0.083 | 16.6 px/sec |

**⚠️ ОШИБКА В ИНТЕРПРЕТАЦИИ (исправлено в v3)**: 
- Первоначальная формула `scaleMovementSpeed` давала **инвертированное** поведение
- При `slow` времени движение было слишком быстрым
- При `fast` времени движение было слишком медленным
- **Решение**: использовать `scaleMovementSpeedInverse` (деление вместо умножения)

#### 2. Кулдауны техник
```typescript
// BASE_COOLDOWN = кулдаун в игровых минутах при normal
// scaledCooldown = BASE_COOLDOWN / (5 / minutesPerTick)

const scaledCooldown = baseCooldown / TIME_SCALING_FACTOR[speed];
```

| Speed | Factor | Пример: BASE=5 мин |
|-------|--------|-------------------|
| superSuperSlow | 20.0 | 0.25 мин (15 сек игры) |
| superSlow | 10.0 | 0.5 мин (30 сек игры) |
| slow | 5.0 | 1 мин игры |
| normal | 1.0 | 5 мин игры |
| fast | 0.33 | 15 мин игры |
| ultra | 0.083 | 60 мин игры (1 час) |

**Интерпретация**:
- При `slow` времени кулдауны короче в игровом времени
- При `fast` времени кулдауны длиннее в игровом времени
- В реальном времени кулдауны ОДИНАКОВЫ (1 тик = 1 сек)

#### 3. Длительность действий
```typescript
// actionDuration в игровых минутах
// realDurationMs = (actionDuration / minutesPerTick) * 1000

const realMs = (actionDurationMinutes / minutesPerTick) * 1000;
```

- LocationScene слушает только `timer:resume` и `timer:pause`
- При старте через `handleMovement()` флаг `isGamePaused` не обновляется

**Решение**:
```typescript
// В setupTickTimer():
this.boundTimerStart = (() => {
  this.onTimerStart();
}) as EventListener;

window.addEventListener('timer:start', this.boundTimerStart);

// Новый метод:
private onTimerStart(): void {
  console.log('[LocationScene] Timer started - resuming physics/tweens');
  this.isGamePaused = false;
  this.physics.resume();
  this.tweens.resumeAll();
}
```

### ФАЗА 2: Интеграция Time Scaling

#### Задача 2.1: Масштабирование скорости игрока

**Файл**: `src/game/scenes/LocationScene.ts`

**Статус**: ⚠️ Исполнено, НО с ошибкой → исправлено в v3

**Что было сделано**:
```typescript
import { scaleMovementSpeed } from '@/lib/game/time-scaling';

const scaledSpeed = scaleMovementSpeed(BASE_PLAYER_SPEED, timeStore.speed);
```

**⚠️ Проблема**:
- Использована неправильная функция
- При `slow` времени: `200 * 5.0 = 1000 px/sec` (слишком быстро!)
- При `fast` времени: `200 * 0.333 = 66 px/sec` (слишком медленно!)

**✅ Исправление в v3**:
```typescript
import { scaleMovementSpeedInverse } from '@/lib/game/time-scaling';

const scaledSpeed = scaleMovementSpeedInverse(BASE_PLAYER_SPEED, timeStore.speed);
// При `slow` времени: `200 / 5.0 = 40 px/sec` (медленно)
// При `fast` времени: `200 / 0.333 = 600 px/sec` (быстро)
```

**Критерии приёмки**:
- [x] Скорость игрока масштабируется при смене time speed
- [❌] ~~При `slow` времени движение быстрее~~ - НАОБОРОТ!
- [❌] ~~При `fast` времени движение медленнее~~ - НАОБОРОТ!
