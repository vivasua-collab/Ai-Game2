## 🚫 ЗАПРЕТ: CADDY НЕ ИСПОЛЬЗУЕТСЯ

**В DEV окружении Caddy НЕ используется!**

WebSocket подключение - **ПРЯМОЕ** к `http://localhost:3003`

```
Browser → localhost:3003 (прямое подключение)
```

### Причина запрета
1. Проект работает только в DEV окружении
2. Нет необходимости в reverse proxy
3. Прямое подключение проще и надёжнее
4. Меньше точек отказа

---

## 🎯 Цель

Проанализировать WebSocket архитектуру и определить путь к унифицированному решению.

---

## 📋 Краткое резюме

### Решение

**Выбрано:** Mini-service на порту 3003 + Socket.IO

**Причины:**
1. Socket.IO ecosystem (rooms, broadcast, fallback)
2. Отдельный порт - независимость от Next.js
3. Знакомый API
4. Fallback polling - работает при проблемах с WebSocket
