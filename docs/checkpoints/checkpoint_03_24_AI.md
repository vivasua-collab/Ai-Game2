## 📋 Обзор

Разработка многоуровневой системы искусственного интеллекта для NPC:

### Уровни AI (по нейротеории)

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI ARCHITECTURE LEVELS                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Level 3: Brain (LLM)        ← Фаза 5+                          │
│  • Диалоги, планирование                                        │
│  • Асинхронно, через Event Bus                                  │
│                                                                  │
│  Level 2: Neural Router       ← Фаза 4+                         │
│  • Маршрутизация сигналов                                       │
│  • Координация между уровнями                                   │
│                                                                  │
│  Level 1: Spinal AI           ← ФАЗА 1-2 (ТЕКУЩИЙ ФОКУС)        │
│  • Рефлексы, уклонение, бегство                                 │
│  • Синхронно на клиенте (< 1мс)                                 │
│  • Классический вариант (правила)                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Классификация NPC

| Тип NPC | AI Уровень | Реализация |
|---------|------------|------------|
| **Временные** (монстры, прохожие) | Spinal AI + State Machine | Фаза 1-2 |
| **Статичные** (торговцы, главы сект) | Spinal AI + GOAP + Brain | Фаза 3+ |

---

## ⚠️ Архитектурное ограничение

**Event Bus асинхронный (10-100мс), но Spinal AI должен быть синхронным (< 1мс)**

**Решение:**
- Spinal AI работает **полностью на клиенте** (в Phaser)
- Event Bus используется **только для критичных данных** (смерть, урон)

См. [checkpoint_03_24_spinal_ai_phase1.md](./checkpoint_03_24_spinal_ai_phase1.md)

## 📝 Примечания

1. **Начать с Фазы 1** - Spinal AI фундамент всей системы

3. **Event Bus только для критичных данных** - смерть, урон
4. **Тестировать производительность** - target < 1мс на update()
5. **Избегать async в update()** - блокирует Phaser
