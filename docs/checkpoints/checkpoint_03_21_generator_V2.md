## ⚠️ ВАЖНО: Обновление формул

**Дата:** 2026-03-21

### Проблема
Текущая реализация использует неверные формулы расчёта урона.

### Корректные формулы

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     ПРАВИЛЬНЫЕ ФОРМУЛЫ V2                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. ЁМКОСТЬ ТЕХНИКИ                                                          │
│     capacity = baseCapacity(type) × 2^(level-1) × masteryBonus             │
│                                                                              │
│  2. УРОН (от ёмкости, НЕ от qiCost!)                                        │
│     finalDamage = capacity × gradeMult                                      │
│                                                                              │
│  3. СТОИМОСТЬ ЦИ                                                             │
│     qiCost = 10 × 1.5^(level-1)                                             │
│     Grade НЕ влияет на qiCost (всегда ×1.0)                                 │
│                                                                              │
│  4. ДЕСТАБИЛИЗАЦИЯ (переполнение ВОЗМОЖНО!)                                 │
│     - Излишки Ци рассеиваются                                               │
│     - Урон практику = excessQi × 0.5                                        │
│     - Урон по цели = inputQi × 0.5 (ТОЛЬКО melee_strike/melee_weapon)      │
│     - Для ranged_*: Ци разлетается, урона по цели НЕТ                       │
│                                                                              │
│  5. ОГРАНИЧЕНИЕ ПО УРОВНЮ (резонанс)                                         │
│     - Техника > уровня практика → НЕ активируется                           │
│     - ЛОР: Ци практика не вытягивает структуру техники                      │
│                                                                              │
│  6. СИСТЕМА ТИКОВ                                                            │
│     - 1 тик = 1 минута игрового времени                                     │
│     - Эффекты рассчитываются в тиках, не в ходах                            │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 🔄 КЛЮЧЕВЫЕ РЕШЕНИЯ (ОБНОВЛЕНО)

### 1. Дестабилизация

**Принято решение:** Переполнение ВОЗМОЖНО, но с последствиями.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ПРАВИЛО ПЕРЕПОЛНЕНИЯ                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  При переполнении техники:                                          │
│                                                                      │
│  1. ИЗЛИШКИ ЦИ РАССЕИВАЮТСЯ                                         │
│     - excessQi = inputQi - capacity                                 │
│     - Рассеивается в окружающее пространство                        │
│                                                                      │
│  2. УРОН ПРАКТИКУ                                                    │
│     - backlashDamage = excessQi × 0.5 (50% от излишков)            │
│                                                                      │
│  3. УРОН ПО ЦЕЛИ — ТОЛЬКО ДЛЯ БЛИЖНЕГО БОЯ!                         │
│     - melee_strike / melee_weapon: effectiveDamage = inputQi × 0.5 │
│     - ranged_*: Ци разлетается во все стороны, урона по цели НЕТ   │
│                                                                      │
│  Примечание:                                                         │
│  Практик не дурак — он знает, какие техники ему подходят.          │
│  Дестабилизация происходит только при ошибках или вынужденных      │
│  ситуациях (например, нет других техник под рукой).                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. Система тиков

**Принято решение:** Эффекты в тиках, не в ходах.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    СИСТЕМА ТИКОВ                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Определение:                                                        │
│  - 1 тик = 1 минута игрового времени                                │
│  - Гибридные бои (не пошаговые)                                     │
│                                                                      │
│  Примеры:                                                            │
│  - Горение: 3 тика = 3 минуты игрового времени                      │
│  - Замедление: 2 тика = 2 минуты игрового времени                   │
│  - Оглушение: 1 тик = 1 минута игрового времени                     │
│                                                                      │
│  Почему тики:                                                        │
│  - Реальное время в бою                                             │
│  - Не зависит от "ходов"                                            │
│  - Синхронизация с другими системами (восстановление Ци и т.д.)    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```typescript
function checkDestabilizationWithBaseQi(
  qiCost: number,
  qiDensity: number,
  capacity: number,
  techniqueSubtype: string
): DestabilizationResult {
  const inputQi = qiCost * qiDensity;
  
  if (inputQi <= capacity) {
    return { isDestabilized: false, effectiveQi: inputQi, efficiency: 1.0 };
  }
  
  const excessQi = inputQi - capacity;
  const isMelee = ['melee_strike', 'melee_weapon'].includes(techniqueSubtype);
  
  return {
    isDestabilized: true,
    effectiveQi: inputQi,
    efficiency: 0.5,
    backlashDamage: Math.floor(excessQi * 0.5),
    targetDamage: isMelee ? Math.floor(inputQi * 0.5) : 0,
  };
}
```
