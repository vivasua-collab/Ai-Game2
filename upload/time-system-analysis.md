# ⏱️ Анализ системы времени: Tick-Based → Auto-Tick

**Дата:** 2026-03-20
**Версия:** 2.0
**Контекст:** Переход от тиковой системы с паузой при неактивности к непрерывным тикам с ручной паузой

---

## 📊 Текущая система (Event-Driven Tick)

### Принцип работы

```
┌─────────────────────────────────────────────────────────────────────┐
│                     EVENT-DRIVEN TIME SYSTEM                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ВРЕМЯ ТЕЧЁТ ТОЛЬКО ПРИ ДЕЙСТВИЯХ                                  │
│                                                                     │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐          │
│   │  Действие   │ ──▶ │   +Тики     │ ──▶ │  time += N  │          │
│   └─────────────┘     └─────────────┘     └─────────────┘          │
│                                                                     │
│   Движение (move_tile)     → +1 тик  (1 минута)                    │
│   Атака (basic_attack)     → +1 тик                                 │
│   Ход боя (combat_turn)    → +2 тика                                │
│   Медитация                → +30-480 тиков                          │
│   Разговор (talk)          → +5 тиков                               │
│                                                                     │
│   ══════════════════════════════════════════════════════════════    │
│                                                                     │
│   ВРЕМЯ НА ПАУЗЕ ПРИ БЕЗДЕЙСТВИИ                                    │
│                                                                     │
│   ⏸️ Просмотр инвентаря   → 0 тиков                                 │
│   ⏸️ Просмотр техник      → 0 тиков                                 │
│   ⏸️ Просмотр статуса     → 0 тиков                                 │
│   ⏸️ Открытие меню        → 0 тиков                                 │
│   ⏸️ БЕЗДЕЙСТВИЕ          → 0 тиков (игрок не делает ничего)        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Достоинства и недостатки

| Достоинство | Описание |
|-------------|----------|
| ✅ Нет фоновых запросов | Сервер опрашивается только при действиях |
| ✅ Нет лаг-проблем | Время не рассинхронизируется |
| ✅ Энергосбережение | Нет лишних вычислений |
| ✅ Простота | Логика простая и понятная |
| ✅ Мультиплеер-готовность | Легко синхронизировать (event-based) |

| Недостаток | Описание |
|------------|----------|
| ❌ Нереалистичность | Мир "замерзает" когда игрок ничего не делает |
| ❌ NPC статичны | NPC не живут своей жизнью без действий игрока |
| ❌ Нет атмосферы | Нет ощущения течения времени |

### Визуализация разницы

```
ВРЕМЯ (линейное)
─────────────────────────────────────────────────────────────────────▶

Текущая система (Event-Driven):
[Действие][ПАУЗА][Действие][ПАУУУУУУУУУУУЗА][Действие][ПАУЗА]
    +1мин         +1мин                       +1мин

[тик][тик][тик][Действие][тик][тик][ПАУЗА][тик][тик][тик]...
  +1мин +1мин +1мин   +1мин  +1мин +1мин         +1мин +1мин +1мин
           ↑
     время течёт само
```

## 🏗️ Архитектура новой системы

### Ядро: AutoTickGenerator

```typescript
// src/lib/game/auto-tick.ts

interface AutoTickConfig {
  /** Интервал тика в мс (по умолчанию 1000 = 1 секунда) */
  tickInterval: number;
  
  /** Авто-пауза при потере фокуса окна */
  pauseOnBlur: boolean;
  
  /** Авто-пауза при открытии диалогов */
  pauseOnDialog: boolean;
}

class AutoTickGenerator {
  private isPaused: boolean = false;
  private intervalId: NodeJS.Timeout | null = null;
  private tickCount: number = 0;
  
  private config: AutoTickConfig = {
    tickInterval: 1000,    // 1 сек = 1 минута игрового
    pauseOnBlur: false,    // не ставить паузу при потере фокуса
    pauseOnDialog: false   // не ставить паузу при диалогах
  };
  
  // Запуск авто-тиков
  start(): void {
    if (this.intervalId) return;
    
    this.intervalId = setInterval(() => {
      if (this.isPaused) return;
      
      this.tickCount++;
      this.emitTick();
    }, this.config.tickInterval);
  }
  
  // Остановка авто-тиков
  stop(): void {
    if (this.intervalId) {
      clearInterval(this.intervalId);
      this.intervalId = null;
    }
  }
  
  // Генерация тика
  private emitTick(): void {
    // Используем СУЩЕСТВУЮЩИЙ обработчик тиков!
    TimeTickService.processTick({
      type: 'auto',
      tickCount: this.tickCount
    });
  }
  
  // Пауза
  pause(): void {
    this.isPaused = true;
    this.emit('tick:pause', { tickCount: this.tickCount });
  }
  
  // Продолжение
  resume(): void {
    this.isPaused = false;
    this.emit('tick:resume', { tickCount: this.tickCount });
  }
  
  // Переключение
  toggle(): void {
    this.isPaused ? this.resume() : this.pause();
    return this.isPaused;
  }
  
  // Получение состояния
  getState(): { isPaused: boolean; tickCount: number } {
    return {
      isPaused: this.isPaused,
      tickCount: this.tickCount
    };
  }
}

// Singleton
export const autoTick = new AutoTickGenerator();
```

## ⚠️ Риски и решения

### Риск 1: Нагрузка от частых тиков

**Проблема:** 1 тик в секунду = 60 запросов в минуту?

**Решение:**
```
❌ НЕ НУЖНО отправлять запрос на каждый тик!

✅ Тики обрабатываются ЛОКАЛЬНО в TruthSystem
✅ Синхронизация с сервером при:
   - Сохранении
   - Критических событиях
   - Смене локации
   - Периодически (каждые 30 сек)
```

### Риск 2: Пауза и мультиплеер

**Проблема:** В мультиплеере пауза одного игрока влияет на других?

**Решение:**
```
Single-player сейчас:
- Пауза глобальная (остановка всех процессов)

Multi-player потом:
- Пауза локальная (только для UI игрока)
- Время на сервере течёт всегда
- При возобновлении — синхронизация с сервером
```

### Риск 3: Потеря игрока

**Проблема:** Игрок ставит паузу и уходит надолго

**Решение:**
```
1. Авто-сохранение при паузе > 5 минут
2. При resume — проверка актуальности данных
3. Уведомление "Вы отсутствовали X минут"
```

## 🔄 Система конвертации времени (Секунды ↔ Тики)

### Проблема

В кодовой базе много констант в секундах:

```typescript
// Примеры существующих констант в секундах
const MEDITATION_DURATION = 1800;      // 30 минут в секундах
const QI_REGEN_INTERVAL = 60;          // 1 минута в секундах
const FATIGUE_TICK_RATE = 300;         // 5 минут в секундах
const COOLDOWN_BASE = 120;             // 2 минуты в секундах
const REST_DURATION = 3600;            // 1 час в секундах
```

При переходе на переменные тики (где 1 тик может быть 1, 5, 10 минут) нужно:
1. Сохранить константы в секундах (источник истины)
2. Конвертировать в тики при использовании
3. Учитывать текущую скорость тика

### TimeConverter API

```typescript
// src/lib/game/time-converter.ts

class TimeConverter {
  private static instance: TimeConverter | null = null;
  private minutesPerTick: number = 1;  // По умолчанию 1 минута за тик

  static getInstance(): TimeConverter {
    if (!TimeConverter.instance) {
      TimeConverter.instance = new TimeConverter();
    }
    return TimeConverter.instance;
  }

  /**
   * Установить скорость тика
   * @param minutes - игровых минут за один тик
   */
  setMinutesPerTick(minutes: number): void {
    this.minutesPerTick = Math.max(1, minutes);
  }

  /**
   * Получить текущую скорость тика
   */
  getMinutesPerTick(): number {
    return this.minutesPerTick;
  }

  /**
   * Конвертировать секунды в тики
   * @param seconds - время в секундах (ИСТОЧНИК ИСТИНЫ)
   * @returns количество тиков (округление вверх)
   */
  secondsToTicks(seconds: number): number {
    const minutes = seconds / 60;
    return Math.ceil(minutes / this.minutesPerTick);
  }

  /**
   * Конвертировать тики в секунды
   * @param ticks - количество тиков
   * @returns время в секундах
   */
  ticksToSeconds(ticks: number): number {
    const minutes = ticks * this.minutesPerTick;
    return minutes * 60;
  }

  /**
   * Конвертировать минуты в тики
   */
  minutesToTicks(minutes: number): number {
    return Math.ceil(minutes / this.minutesPerTick);
  }

  /**
   * Конвертировать тики в минуты
   */
  ticksToMinutes(ticks: number): number {
    return ticks * this.minutesPerTick;
  }

  /**
   * Получить человекочитаемую длительность
   */
  formatTickDuration(ticks: number): string {
    const minutes = this.ticksToMinutes(ticks);
    const hours = Math.floor(minutes / 60);
    const mins = minutes % 60;

    if (hours > 0 && mins > 0) {
      return `${hours}ч ${mins}м`;
    } else if (hours > 0) {
      return `${hours}ч`;
    } else {
      return `${mins}м`;
    }
  }
}

export const timeConverter = TimeConverter.getInstance();
```
